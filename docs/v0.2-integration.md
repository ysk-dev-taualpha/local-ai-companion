# v0.2 Go-Python 結合確認手順

## 概要

v0.2 では Go Runtime が Python AI Service へ HTTP で会話リクエストを転送します。
このドキュメントでは両サービスを手動起動し、動作を確認する手順を説明します。

## 前提

- Python 3.x
- Go 1.21+
- リポジトリを clone 済み

## 起動手順

### 1. Python AI Service を起動

```bash
cd local-ai-companion
PYTHONPATH=./src python3 -m local_ai_companion --serve
```

起動ログ:
```
Python AI Service listening on 127.0.0.1:8090
```

### 2. Go Runtime を起動

別のターミナルで:

```bash
cd local-ai-companion
go run ./cmd/local-ai-runtime --config config.runtime.example.json
```

起動ログ:
```
[INFO] Go Runtime starting on 127.0.0.1:8080
[INFO] Python AI Service URL: http://127.0.0.1:8090
[INFO] Go Runtime ready
```

`python_service.command` を設定すると、Go Runtime が Python AI Service を子プロセスとして起動し、`python_service.base_url` に到達できるまで待ってから Runtime を公開します。

```json
{
  "python_service": {
    "base_url": "http://127.0.0.1:8090",
    "command": "PYTHONPATH=./src python3 -m local_ai_companion --serve",
    "ready_timeout_ms": 10000,
    "shutdown_timeout_ms": 5000
  }
}
```

この場合、上記の「Python AI Service を起動」は不要です。Go Runtime の graceful shutdown 時に子プロセスも停止します。

## 動作確認

### Health Check

```bash
curl http://127.0.0.1:8080/healthz
# {"status":"ok"}
```

### Python AI Service 直接呼び出し

```bash
curl -X POST http://127.0.0.1:8090/v1/conversation \
  -H "Content-Type: application/json" \
  -d '{"message":"こんにちは","request_id":"test-1"}'
```

期待される応答 (mock provider):
```json
{
  "request_id": "test-1",
  "conversation_id": "default",
  "assistant": {
    "text": "受け取りました: こんにちは",
    "emotion": "neutral",
    "motion": "nod",
    "speak_style": "normal",
    "interruptible": true
  }
}
```

### Go Runtime 経由での呼び出し

```bash
curl -X POST http://127.0.0.1:8080/v1/conversation \
  -H "Content-Type: application/json" \
  -d '{"message":"こんにちは","request_id":"test-2"}'
```

Go Runtime が Python AI Service にリクエストを転送し、応答を返します。
期待される応答は直接呼び出しと同じ形式です。

## エラーケース確認

### Python Service 未起動時のエラー

Go Runtime のみ起動している状態で:

```bash
curl -X POST http://127.0.0.1:8080/v1/conversation \
  -H "Content-Type: application/json" \
  -d '{"message":"hi"}'
```

応答:
```json
{"error":{"code":"python_service_error","message":"python service unavailable: ..."}}
```

### 無効なリクエスト

```bash
curl -X POST http://127.0.0.1:8080/v1/conversation \
  -H "Content-Type: application/json" \
  -d '{}'
```

応答:
```json
{"error":{"code":"invalid_request","message":"message is required"}}
```

```
User / curl
  │
  ▼ POST /v1/conversation
Go Runtime (:8080)
  │ HTTP POST /v1/conversation
  ▼
Python AI Service (:8090)
  │ ConversationCore.send()
  ▼
Mock / OpenAI LLM
```
